# 实例页创建弹窗设计

**目标**

将实例页现有的内联创建表单改为复用弹窗组件，聚焦“按节点多选创建实例”的主路径，同时优化实例列表操作区稳定性与复制操作效率。

**范围**

- 实例页顶部保留工具栏，新增“创建实例”入口，点击后通过服务端返回弹窗 HTML。
- 创建弹窗仅保留：
  - 订阅过滤，支持“全部订阅”
  - 节点多选
  - `autoSwitch` 开关，默认关
- 创建行为默认 `autoStart=true`，移除显式“自动启动”选项。
- 移除“数量”和“批量创建”表单。
- 节点选择区域改为卡片网格布局，桌面端一行 4 列。
- 实例列表操作区增加稳定宽度，避免“检测”切换到“检测中...”时布局左右抖动。
- 实例检测触发后恢复滚动位置，避免列表刷新后回到顶部。
- “复制链接”改为两个直接复制按钮：`复制 SOCKS5`、`复制 HTTP`。

**实现方式**

- 继续沿用 HTMX 服务端渲染模式，不引入新的前端框架。
- 复用现有 `renderUIModal(...)`，新增实例创建弹窗渲染函数和 GET 路由。
- 创建表单继续复用现有 `/ui/action/instances/create`，但将输入模型收敛为“节点多选为主”，后端默认 `autoStart=true`、`autoSwitch=false`。
- 订阅过滤优先在弹窗内通过服务端渲染切换，避免把所有节点筛选逻辑堆到前端。
- 滚动位置恢复放在 `htmx.js` 中统一处理，仅对刷新 `#ui-tab` 的实例页操作启用。

**不做**

- 不调整底层 `/api/instances` 接口契约。
- 不引入分页、虚拟列表或更复杂的前端状态管理。
- 不修改订阅页和代理池页的交互形态。
