# 管理页统一下拉组件设计

**目标**

将管理页当前所有原生 `<select>` 替换为统一的 `ui-select` 组件，解决原生下拉在弹窗和页面中展开层不可控、样式不统一、动画不可控的问题。

**范围**

- 替换当前管理页中已存在的 6 处原生下拉：
  - 设置页：`allowLan`
  - 设置页：`logLevel`
  - 设置页：`releaseChannel`
  - 设置页：`proxyAuthEnabled`
  - 创建实例弹窗：`subscriptionId`
  - 创建实例弹窗：`autoSwitch`
- 保持原有表单字段名和后端提交契约不变。
- 保持 HTMX 提交、自动筛选和现有业务逻辑不变。

**不做**

- 不引入前端框架。
- 不改 API 或表单参数名。
- 不额外做搜索型下拉、多选下拉或虚拟列表。

**根因**

当前问题不是某一个弹窗的层级写错，而是原生 `<select>` 的展开菜单由浏览器/系统控件绘制，不是普通 DOM 弹层。结果是：

- 展开位置和覆盖关系不可精确控制
- 在弹窗内看起来像“选项容器压住主控件”
- 无法统一动画、边框和高亮状态

因此要从根上解决，必须改成自定义下拉组件。

**组件方案**

- 结构采用：
  - 容器：`ui-select`
  - 隐藏域：真正提交的 `<input type="hidden" name="...">`
  - 触发器：按钮，显示当前选中标签
  - 菜单：绝对定位的 `ui-select-menu`
  - 选项：按钮式 `ui-select-option`
- 组件支持：
  - 单选
  - 默认值显示
  - 点击外部关闭
  - 选项切换后更新隐藏域
  - 可选的“切换后自动提交表单”

**交互规则**

- 点击触发器展开或收起菜单。
- 点击选项后：
  - 更新隐藏域值
  - 更新触发器文本
  - 更新选中态
  - 若配置了自动提交，则调用 `form.requestSubmit()`
- 同一时刻只允许一个下拉展开。
- 按 `Escape` 或点击外部关闭。

**实现方式**

- `ui_htmx.go`
  - 抽出统一 `renderUISelect(...)` helper，避免 6 处重复拼 HTML。
  - 服务器渲染初始选中状态和标签。
- `htmx.js`
  - 新增统一下拉交互逻辑，使用事件委托处理开关、选中、自动提交与外部关闭。
- `style.css`
  - 新增 `ui-select`、触发器、菜单、选项的统一样式与过渡动画。
  - 菜单使用普通 DOM 层，保证在 modal 内也可控。

**验收标准**

- 页面中不再输出这 6 处原生 `<select>`。
- 下拉菜单不再覆盖主控件区域，而是作为自定义 popover 展开。
- 创建实例弹窗与设置页的下拉视觉和交互一致。
- 切换“过滤订阅”后仍会自动刷新当前弹窗内容。
