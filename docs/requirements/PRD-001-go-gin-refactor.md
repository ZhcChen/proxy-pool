# PRD-001：Bun 全量迁移为 Go + Gin + HTML/HTMX

## 1. 背景

当前项目后端基于 Bun（TypeScript），前端为单页 JS。随着实例调度、订阅解析、健康检查、OpenAPI、代理认证等能力持续累积，继续在 Bun 体系扩展的维护成本变高。

本次重构不是“选几块迁移”，而是以当前 Bun 版本为基线，执行**全量功能迁移**：后端迁到 Go，页面重做为 HTML + HTMX，并保持线上行为一致。

## 2. 重构原则

1. 全量迁移：现有功能全部盘点、全部迁移，不允许功能缩水。
2. 行为兼容优先：先确保接口、状态码、数据兼容，再做内部实现重构。
3. 部署简化：页面和静态资源随二进制打包，容器内仅运行一个 Go 可执行文件。
4. 运行规范不变：本地开发、联调、测试统一使用 `docker compose`。

## 3. 原功能基线（必须迁移）

### 3.1 后端能力基线

1. 鉴权与会话：`ADMIN_TOKEN` 登录、Bearer 鉴权、`OPENAPI_TOKEN` 独立鉴权。
2. 系统与设置：IP 探测、公网 IP 自动获取、内核状态/版本查询/安装、代理认证重置、运行参数保存。
3. 订阅管理：新增、编辑、更新、刷新、删除、节点列表、节点可用性检测、可用性统计。
4. 实例管理：单个创建、批量创建、多选节点创建、启动、停止、删除、日志、健康检测、`autoSwitch` 编辑。
5. 代理池导出：`/api/pool` 与 `/openapi/pool` 输出实例池地址。
6. 后台任务：autoStart、周期健康检查、订阅节点检查器复用。
7. 存储：`state.sqlite(kv)` + 订阅 YAML 快照 + 实例配置目录 + 内核安装信息。

### 3.2 前端能力基线

1. Token 登录页。
2. 实例页：创建、批量创建、多选创建、实例操作、复制链接、日志弹窗。
3. 订阅页：新增、编辑、更新订阅、删除、节点弹窗、检测单个/全部。
4. 设置页：内核管理、网络参数、代理认证、导出 Host、公网 IP 探测。
5. 代理池页：列表展示与一键复制。

## 4. 目标

1. 后端迁移到 Go 1.25 + Gin。
2. 页面改造为纯 HTML + HTMX 交互。
3. `htmx` 本地引用（仓库内静态文件），禁止 CDN 依赖。
4. Web 资源通过 `go:embed` 打包进二进制，单文件运行。
5. Go 工程目录按 `src` 风格设计，便于长期维护。
6. 复用 `data/` 目录，保证平滑升级与可回滚。
7. Go 侧数据库继续使用 SQLite，并采用纯 Go 驱动（`database/sql + modernc.org/sqlite`）。

## 5. 非目标

1. 本阶段不新增新业务能力（如新调度策略、分布式控制面等）。
2. 本阶段不扩大 OpenAPI 边界（仍仅开放 `GET /openapi/pool`）。
3. 本阶段不做跨节点高可用部署改造。

## 6. 关键兼容要求

1. 鉴权模式保持不变：
   - 管理端：`ADMIN_TOKEN`
   - OpenAPI：`OPENAPI_TOKEN`（独立 token）
2. 数据兼容保持不变：
   - `kv(state)`
   - `kv(mihomo_install)`
   - `kv(proxy_health:{subscriptionId})`
3. 订阅 fallback 逻辑保持不变：
   - 原始 URL
   - `flag=clash-meta`
   - `flag=meta`
   - `flag=clash`
4. 订阅 URL 自动持久化有效 flag 行为保持不变。
5. 实例与节点关系保持不变：一个实例一个 `mixed-port`，对应一个主要出口节点。
6. 开发/测试流程保持 `docker compose`。
7. 保证 SQLite 文件兼容性：可直接复用现有 `data/state.sqlite`。

## 7. 验收矩阵

| 验收项 | 描述 | 验收标准 |
| --- | --- | --- |
| AC-01 | 容器启动 | `docker compose up -d --build` 后 `proxy-pool` 运行正常 |
| AC-02 | 单二进制 | 仅运行 Go 二进制即可访问页面与 API |
| AC-03 | 静态资源打包 | 无外部 `web` 目录依赖，模板/资产由 embed 提供 |
| AC-04 | HTMX 本地化 | 页面仅引用本地 `htmx.min.js`，无 CDN 链接 |
| AC-05 | 登录鉴权 | `/api/login` 与管理鉴权行为与现网一致 |
| AC-06 | OpenAPI 鉴权 | `OPENAPI_TOKEN` 独立生效，`ADMIN_TOKEN` 不能调用 `/openapi/pool` |
| AC-07 | 设置管理 | 设置保存、重置代理认证、公网 IP 探测功能可用 |
| AC-08 | 内核管理 | 版本查询、安装、状态查询行为正确 |
| AC-09 | 订阅新增 | URL/YAML 新增订阅、快照写入正常 |
| AC-10 | 订阅更新 | 编辑与“更新订阅”行为正确，fallback 与 flag 持久化正确 |
| AC-11 | 订阅删除 | 被引用时拒删；可删时清理 YAML 与健康缓存 |
| AC-12 | 节点检测 | 订阅节点单测/全测与可用性统计正确 |
| AC-13 | 实例创建 | 单个、批量、多选创建与端口分配逻辑正确 |
| AC-14 | 实例运维 | 启动、停止、删除、日志、检测、autoSwitch 编辑可用 |
| AC-15 | 自动任务 | autoStart 与周期健康检查稳定运行 |
| AC-16 | 代理池导出 | `/api/pool` 与 `/openapi/pool` 输出正确 |
| AC-17 | 数据兼容 | 复用既有 `data/state.sqlite` 后数据可读 |
| AC-18 | 测试门禁 | `tests/go` 迁移后全部通过，并补齐新增回归用例 |
| AC-19 | 回滚能力 | 失败可按文档回滚到 Bun 稳定版本 |
| AC-20 | 构建约束 | Go 版本可在 `CGO_ENABLED=0` 下构建运行（无 C 依赖） |

## 8. 里程碑

1. M1：完成 Bun 功能基线冻结与 Go `src` 目录骨架。
2. M2：完成 JSON API 全量迁移与存储/运行时兼容。
3. M3：完成 HTML + HTMX 页面全量迁移与 embed 打包。
4. M4：完成容器切换、回归测试、上线与回滚预案。

## 9. 风险与缓解

1. 功能遗漏风险：先建立“接口与页面功能矩阵”，逐项对照验收。
2. 进程管理差异：增加端口清理与僵尸进程回收测试。
3. 页面交互回归：关键流程做 HTMX 片段级 smoke 覆盖。
4. 数据兼容风险：启动阶段加入状态反序列化校验与告警。
5. SQLite 驱动风险：纯 Go 驱动若出现兼容/性能问题，预留切换到 `go-sqlite3` 的回退方案。

## 10. 确认点

1. 以当前 Bun 代码行为作为迁移基线，按“全量功能等价”推进。
2. Go 工程目录采用 `src` 风格组织（详见 SPEC）。
